2. Вводная часть

Актуальность темы

ОС как фундамент: Уязвимости ОС сводят на нет внешние периметры безопасности (ГОСТ Р 50922-2006. Защита информации. Основные термины и определения 2007 г.).
Эскалация привилегий: Критически важная фаза любой APT-атаки.
Shatter attack: Уязвимость из-за отсутствия мандатного контроля в графической подсистеме.
Значимость: Понимание векторов атак на IPC необходимо для построения надежных систем.

Цели и задачи работы

Цель: Изучить архитектурную уязвимость оконной подсистемы ОС Windows и механизмы защиты.

Задачи:

Рассмотреть эскалацию привилегий и изоляцию процессов. Проанализировать архитектуру Win32 API. Разобрать алгоритм классической Shatter-атаки. Изучить механизмы защиты (Session 0 Isolation, MIC/UIPI).

3. Основная часть

Основы эскалации привилегий

Эскалация привилегий — несанкционированное повышение прав доступа субъектом (Сухомлин 2021 г.).

Основные виды:

Вертикальная: Получение прав админа или ядра (SYSTEM) обычным юзером.

Горизонтальная: Получение прав другого юзера того же уровня (например, кража сессий).

Изоляция и кольца защиты

Основа защиты ОС — аппаратные кольца (Protection Rings) (Кулябов и Королькова 2018 г.).

Ring 0 (Ядро): Полный доступ к железу (службы и драйверы).

Ring 3 (User-mode): Изолированные приложения.

Переход между ними идет строго через системные вызовы (см. рис. 1).

Обмен сообщениями (Win32 API)

Графический интерфейс управляется событиями (Руссинович и др. 2018 г.).

Общение окон идет через вызовы SendMessage и PostMessage.

Фундаментальная проблема: Старые ОС не проверяли уровень привилегий отправителя к получателю на одном рабочем столе.

Механизм Shatter-атаки

Атака позволяет выполнить произвольный код с правами SYSTEM.

Этапы реализации:

Разведка: Поиск окна системной службы.

Внедрение: Загрузка шелл-кода в процесс-жертву (через WM_SETTEXT).

Триггер: Отправка сообщения с адресом шелл-кода (например, WM_TIMER).

Выполнение: Привилегированный процесс передает управление коду.

Session 0 Isolation (с Windows Vista)

Проблема: Службы и первый юзер сидели в общем Сеансе 0.

Решение: Сеанс 0 жестко изолирован только для системных служб.

Итог: Пользовательские процессы физически не видят окна системных служб (см. рис. 3).

User Interface Privilege Isolation (UIPI)

(Надстройка над механизмом MIC (Малюк 2020 г.))

Уровни целостности: Low, Medium, High, System.

Правило: Процесс с низким уровнем не может слать сообщения окну с высоким.

Итог: Блокировка Shatter-атак внутри одного сеанса (см. рис. 4).

4. Заключение

Вывод

Недочеты в механизмах IPC могут привести к полной компрометации ядра (Ring 0). Классическая Shatter-атака доказала слабость дискреционного подхода (DAC). Современная защита опирается на строгий мандатный контроль (MAC) (Шаньгин 2022 г.). Надежная безопасность требует комплексной изоляции на всех уровнях.
